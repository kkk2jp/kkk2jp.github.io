## プロンプトエンジニアリングとは

LLM（大規模言語モデル）に対して、意図した出力を得るためのプロンプト設計技術です。

## 基本テクニック

### 役割指定（Role Prompting）

```
あなたはシニアのソフトウェアエンジニアです。
以下のコードをレビューしてください。
```

### Few-shot Prompting

```
入力: 「りんご」→ 出力: 「果物」
入力: 「犬」→ 出力: 「動物」
入力: 「バラ」→ 出力: ?
```

### Chain-of-Thought

```
問題を解く前に、ステップバイステップで考えてください。
まず...次に...最後に...という形式で答えてください。
```

## 構造化出力の指定

```
以下をJSON形式で出力してください：
{
  "title": "記事タイトル",
  "summary": "200文字以内の要約",
  "tags": ["タグ1", "タグ2"]
}
```

## まとめ

明確な指示・具体的な例・出力形式の指定の3つを意識するだけで、LLMの出力品質が大幅に向上します。
